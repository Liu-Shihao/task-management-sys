# FinBlock 任务管理系统 - 需求分析

## 1. 问题陈述 (Problem Statement)

**核心问题**：FinBlock 用户需要一种高效的方式来管理和执行部署任务，目前缺乏系统化的任务管理和自动化执行能力。

**当前替代方案**：手动管理 Excel 表格、逐个登录 Jenkins/Ansible 触发任务

**触发原因**：用户希望简化部署流程，将任务管理、模板复用、自动化执行整合到一个系统中

---

## 2. 需求层级 (Need Hierarchy)

| 层级 | 需求 |
|------|------|
| **核心需求** | 批量导入任务数据、管理部署任务、自动化执行 |
| **功能需求** | Excel 上传、模板管理、任务创建、任务执行 |
| **非功能需求** | 实时状态更新、错误可追溯 |

---

## 3. 用户故事分析

### Story 1: Excel File Upload

| 要素 | 内容 |
|------|------|
| **用户** | FinBlock 用户 |
| **动机** | 简化数据录入流程 |
| **期望结果** | 直接上传 Excel 生成任务 |

**验收标准分析**：
- 支持 .xlsx, .xls 格式
- 文件类型校验
- 解析错误需明确提示具体问题

### Story 2: Template Management

| 要素 | 内容 |
|------|------|
| **用户** | FinBlock 用户 |
| **动机** | 复用部署配置 |
| **期望结果** | 快速创建基于模板的任务列表 |

**验收标准分析**：
- 创建新模板
- 克隆已有模板
- 模板可保存和复用

### Story 3: Release Rundown Creation

| 要素 | 内容 |
|------|------|
| **用户** | FinBlock 用户 |
| **动机** | 快速生成部署任务清单 |
| **期望结果** | 基于模板的任务序列 |

**验收标准分析**：
- 选择模板生成 rundown
- 唯一标识 + Pending 状态
- 展示任务列表、顺序、初始状态

### Story 4: Task Controls

| 要素 | 内容 |
|------|------|
| **用户** | FinBlock 用户 |
| **动机** | 灵活管理单个任务 |
| **期望结果** | 可编辑、可删除、可执行 |

**验收标准分析**：
- Edit: 修改任务参数
- Delete: 确认后删除
- Run: 触发自动化执行

### Story 5: Automated Execution

| 要素 | 内容 |
|------|------|
| **用户** | FinBlock 用户 |
| **动机** | 无需手动触发部署 |
| **期望结果** | 一键触发 Jenkins/Ansible |

**验收标准分析**：
- 支持 Jenkins Pipeline 触发
- 支持 Ansible Job 触发
- 实时状态更新（In Progress/Completed/Failed）
- 展示构建/作业 URL
- 失败时可查看错误日志

## 5. 假设 (Assumptions)

1. Jenkins 和 Ansible 已经部署完成，有 API 接口可供调用
2. Excel 模板格式已定义或有标准规范
3. 用户有 Jenkins/Ansible 的认证凭证
4. 系统需要用户认证

---

## 6. 范围定义 (Scope)

### MVP (V1)

- [x] Excel 文件上传和解析
- [x] 模板创建和克隆
- [x] Release Rundown 创建
- [x] 任务的基本 CRUD
- [x] Jenkins/Ansible 触发
- [x] 任务状态同步

### V2 (暂不包含)

- 任务依赖关系管理
- 定时执行
- 权限管理
- 审计日志
- 通知系统
